# Personal Multimodal RAG Planner

Personal study-planning agent with multimodal retrieval over course documents, planner screenshots, study-location photos, to-do lists, and external research papers. The agent is implemented with LangGraph, OpenAI chat models, OpenCLIP embeddings, ChromaDB vector storage, and BM25 lexical retrieval.

The submitted system supports six planner capabilities:

- C1 CourseQA / TopicSummary over course-content documents.
- C2 Study-duration estimation from course evidence.
- C3 Free-slot finding from planner screenshots.
- C4 Study-environment recommendation.
- C5 Targeted study-environment fit assessment.
- C6 ResearchQA over the external-knowledge paper corpus.

## Repository Layout

- `planner_agent.py`: LangGraph planner agent, tools, retrieval, multimodal prompting, and CLI loop.
- `config.py`: runtime settings and RAG ablation modes.
- `text_tokenization.py`: shared BM25 tokenization logic.
- `scripts/`: ingestion, chunking, PDF rendering, and Chroma indexing scripts.
- `data/kb/metadata/`: CSV/JSONL metadata for documents, courses, assignments, tasks, environments, and research papers.
- `data/kb/00_raw/`: source KB documents and images.
- `data/kb/01_processed/`: extracted text, chunks, and rendered PDF page images.
- `data/kb/02_index/`: BM25 corpus files.
- `chroma_db/`: local persistent Chroma vector store generated by ingestion.
- `eval/`: benchmark cases and evaluation runner.
- `requirements.txt`: Python dependencies.
- `.env.example`: environment variable template.

## Installation

Use Python 3.12 if possible.

```bash
cd personal-multimodal-RAG-planner
python3 -m venv .venv
source .venv/bin/activate
python -m pip install --upgrade pip
pip install -r requirements.txt
cp .env.example .env
```

Edit `.env` and set at least:

```bash
OPENAI_API_KEY="your_api_key_here"
OPENAI_MODEL="gpt-5.4-mini"
PLANNER_RAG_MODE="text_retrieval_mllm"
```

If OCR is needed for image documents, install the Tesseract system binary as well. On macOS:

```bash
brew install tesseract
```

## Build Or Refresh The Indexes

The agent expects the processed KB, BM25 corpus, and Chroma indexes to exist. If `data/kb/01_processed/`, `data/kb/02_index/`, or `chroma_db/` are missing or stale, rebuild them:

```bash
source .venv/bin/activate
bash scripts/run_full_ingestion.sh
```

Equivalent step-by-step commands:

```bash
python scripts/extract_documents.py
python scripts/render_pdf_pages.py
python scripts/recursive_chunk_documents.py
python scripts/index_text_chunks.py
python scripts/index_pdf_page_images.py
python scripts/index_raw_images.py
```

Changing `OPEN_CLIP_MODEL`, `OPEN_CLIP_PRETRAINED`, `TEXT_COLLECTION_NAME`, or `TEXT_CONTEXT_MODE` requires rebuilding the relevant indexes.

## Run The Agent

Interactive CLI:

```bash
source .venv/bin/activate
python planner_agent.py
```

Useful Makefile targets:

```bash
make ingest
make run
make run-eval
make ablate-multimodal
make ablate-filter
make ablate-all
```

## RAG Modes

Set `PLANNER_RAG_MODE` in `.env` or inline:

```bash
PLANNER_RAG_MODE=text_only python planner_agent.py
PLANNER_RAG_MODE=text_retrieval_mllm python planner_agent.py
PLANNER_RAG_MODE=multimodal_retrieval_mllm python planner_agent.py
```

Modes:

- `text_only`: dense + BM25 text retrieval only.
- `text_retrieval_mllm`: text retrieval plus available images passed to the multimodal model for understanding.
- `multimodal_retrieval_mllm`: text retrieval plus image-index fusion, with images passed to the multimodal model.

Metadata filtering can be toggled with:

```bash
RETRIEVAL_METADATA_FILTER_ENABLED=0 python planner_agent.py
RETRIEVAL_METADATA_FILTER_ENABLED=1 python planner_agent.py
```

## Evaluation

Self-test the evaluation utilities:

```bash
python eval/run_benchmark.py --self-test
```

Run a smoke benchmark:

```bash
python eval/run_benchmark.py --smoke
```

Run one variant:

```bash
python eval/run_benchmark.py --variant multimodal_retrieval_mllm_filter_on
```

Full benchmark results are written to `eval/results/`:

- `runs.jsonl`
- `summary_by_variant.csv`
- `summary_by_family.csv`
- `report_tables.md`
